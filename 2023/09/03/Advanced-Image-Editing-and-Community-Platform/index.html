Advanced Image Editing and Community Platform

发表于2023-09-03|更新于2023-09-04

  1. What problem I am trying to solve?
    创造一个图像处理网站旨在解决与图像编辑、用户管理、社区参与、数据处理、安全性和用户体验相关的各种挑战。它提供了一个平台，让用户能够编辑和分享图像，同时确保其数据安全且互动顺畅。

  2. Why is this important?

  • 综合性：网站提供了一个综合性的解决方案，包括图像处理、社交互动和个人图像库功能。这使得用户可以在一个平台上完成多种任务，而不必在不同的网站之间切换。
  • 完整性：具备了一个网站基本的要素。能够使用jwt，md5，数据库提供用户认证，数据安全和持久化存储。
  • 用户友好的界面：如果您的网站有一个直观、易于使用的界面，可以减轻用户的学习曲线，这是一个竞争优势。
  • 扩展性：学习图像处理相关技术后，都可以在原本的网页功能上进行进一步的扩展。

  1. How do I understand this problem and how do I formulate it into a math/CS problem?
    主要的是将页面上的各种信息转化为数据库内表的问题。
    评论数据库表包括了以下字段：

    • id：这是一个自动生成的递增主键，用于唯一标识每个评论记录。
    • content：这个字段存储评论的内容，通常是文本。
    • create_time：这是评论记录创建的时间戳，用于跟踪评论的时间顺序。
    • parent_message：这是一个指向父评论的外键，用于建立评论的层次结构（回复评论）。
    • publisher：这是一个外键，关联到发布评论的用户。
    • picture：这是一个外键，关联到评论中可能包含的图片或多媒体内容。

    同样的有用户管理模块的数据库设计：

    • Username是主键，是用户创建的账号。
    • Nickname是用户昵称，用户可以修改。
    • Password是用户密码，用户可以修改。
    • Avatar是头像存储的路径。

具体做了啥？

Implemented a front-end and back-end separated image processing website based on Python framework django
which is based on Model-Template-View design pattern.

  • Established a basic account system, including registration functions, password modification functions, etc.Users can
    log in, upload images, crop and doodle the images and finally stored the modified images
  • Established community functions. Users can share their photos with others, who may comment.
  • Used xmlhttp to send front-end data to the backend
  • Used SQL Database.All useful information is stored in the database,where password is encrypted with MD5
  • Used jwt token for user identity verification
