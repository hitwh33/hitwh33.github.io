Efficient Helmet Detection with OpenVINO and YOLO

发表于2023-09-03|更新于2023-09-04

  1. What problem I am trying to solve?
    借助yolo和openvino工具包，使用cpu完成对头盔检测的推理。相比gpu，更能在低功耗，环境恶劣的情况下工作。

  2. Why is this important?
    对于社会：电动车作为一种常用的交通工具但却比较危险。驾乘人员如果佩戴安全头盔，头盔光滑的半球形外壳和内部泡沫缓冲层，可有效避免颅骨遭受撞击。
    对于我们：帮助我们熟悉并且掌握openvino工作套件的使用。一起组队完成任务，能加强组员的团队合作能力。建立区别于学校小实验的项目思维，在以后的工程项目中大有用处。

  3. What’s the current solution? And my solution?
    当前市面上软件面临的问题：
    当前的解决方案可能涉及使用GPU进行头盔检测的推理，需要考虑电源供应和设备的便携性。同时在尘土多的环境中，GPU相对于CPU更容易受到影响和损坏。所以GPU不合适。

    我的方案：
    我的解决方案是利用OpenVINO工具包，使用CPU进行头盔检测的推理。这种解决方案在低功耗、环境恶劣的情况下更为适用，因为CPU通常比GPU更节能，而且不需要额外的散热系统。OpenVINO还提供了优化和加速的功能，使得在CPU上运行的推理速度可以达到可接受的水平

  1. What challenges do I met and how do I solve the problems
    OpenVINO工具包的学习曲线： 不熟悉OpenVINO工具包，学习如何配置、优化和部署模型可能需要时间。解决方法是查阅OpenVINO文档、参考教程和示例代码，以逐步了解其功能和用法。

  2. 项目步骤：

  • 数据集处理：
    1.首先将原始数据集标签文件的VOC格式(xml)转换为YOLO格式(txt)文件方便训练。（这就需要做一步坐标转换，将左上与右下坐标转换为中心坐标与相对宽高）
    2.对数据集进行划分，分为20%验证集和80%训练集。

  • 数据集处理之数据增广：
    我们使用yolov5自带的数据增广功能，albumentations.
    在utils文件中查看augmentations.py

  • 数据模型训练：

    背景的“误识别”主要是由于数据集的标注不全产生的有些“背景”是未被标注的人的头部，有些无头盔的头部则会因为数据集中存在未被标注的无头盔的头部而被误判为背景

  • 模型转换：

    1.先将预训练的结果pt文件使用yolo自带的export.py转换为onnx文件。
    2.随后导入Netron查看网络结构，
    发现共三个Transpose层，其上的第一个卷积层分别为Conv_139，Conv_157，Conv_175。

    1. 随后运行openVINO的模型优化器转化为IR推理模型

  • 模型预测：

  • 性能测评：

  • OpenVINO:

    对于原始的模型没有做任何参数数量级上的轻量化的情况下，对于一个30帧的视频检测速度纯CPU可以达到20.37FPS，平均49.08ms.

    此外内存占用仅170MB左右.

    tensorrt推理时间更短，4ms。但由于需要初始化cuda和cudnn，显存最高占用到1787MB

    对比结果：
    我们可以看到tensorrt在速度上占有了一些优势，但是这是一块功耗240瓦左右的GPU，而我们的open vino可以在低功耗的条件下达到与之相近的效能与推理精度，openvino在内存占用和功耗上都占有绝对优势，且其帧率对于边缘设备来讲也是完全够用的，因此使用open vino作为推理引擎对我们项目来说更为合适，综合考虑上更适合我们项目应用的较恶劣环境。

具体做了啥：
• collected and sorted a dataset of 764 Helmet Detection instances from Tianchi Datasets.
• Transformed dataset labels from VOC format (XML) to YOLO format (txt) for training simplicity.
• Utilized Albumentations for data augmentation in object detection.
• Optimized the model by converting the trained pt file to ONNX and further into an IR inference model using Yolo’s
export.py and OpenVINO, achieving a remarkable CPU-based detection speed of 20.37 FPS for a video.
• Reduced memory footprint to a mere 170MB when utilizing only the CPU, making it ideal forresource-intensive environments compared to TensorRT’s 1787MB GPU memory usage.
